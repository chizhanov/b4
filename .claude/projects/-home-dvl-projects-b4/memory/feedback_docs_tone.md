Documentation tone must be factual and explanatory, not marketing/promotional. b4 is free — no need to "sell" it. Avoid words like "удобный", "продвинутый", superlatives, or feature bullet lists that read like a landing page. Just explain how things work for non-technical users.

**Why:** User explicitly requested this — the docs are for people who already chose to use b4, not for convincing someone to try it.

**How to apply:** When writing any docs content, keep it dry and informative. Describe what something does and how to use it, not why it's great.
