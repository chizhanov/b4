# Маршрутизация

Вкладка маршрутизации содержит два раздела: **DNS-редирект** и **Маршрутизация трафика**.

## DNS-редирект

Перенаправляет DNS-запросы для доменов из сета на указанный DNS-сервер. Используется, когда провайдер подменяет DNS-ответы (DNS poisoning).

### Настройка

1. Включите **DNS-редирект**
2. Выберите DNS-сервер из списка или укажите IP вручную

Доступны популярные DNS-серверы с пометками:

- **Fast** — ориентированы на скорость
- **AdBlock** — блокируют рекламу
- **DNSSEC** — поддерживают DNSSEC

### Фрагментировать запрос

Разбивает DNS-запрос на части. Полезно, если провайдер анализирует и подменяет DNS-пакеты даже к сторонним серверам.

## Маршрутизация трафика

Направляет трафик, совпавший с целями сета, через указанный сетевой интерфейс. Используется для маршрутизации через VPN, WireGuard или другой туннель.

### Настройка

1. Включите **Маршрутизацию**
2. Выберите **Исходные интерфейсы** — с каких интерфейсов обрабатывать трафик (если не выбраны — со всех)
3. Выберите **Выходной интерфейс** — куда направить трафик (например, `wg0` для WireGuard)

### IP TTL

Время жизни пакета (секунды). Управляет, как долго маршрутизируемый трафик будет ходить через указанный интерфейс.
